## 🔒 反幻觉机制 (新增强制要求)

### 1. 数据溯源 (Data Provenance)

- **来源要求**: 所有数据必须来自权威机构
  - ✅ 国家统计局、央行、财政部等官方数据
  - ✅ McKinsey/BCG/Deloitte/EY 等权威咨询报告
  - ✅ 上市公司财报、交易所公告
  - ✅ 行业协会官方统计
  - ❌ 禁止使用"据估计"、"约"、"可能"等模糊表述
  - ❌ 禁止使用无法追溯来源的数据

- **标注格式**: 每个数据必须标注

```markdown
[数据内容] (来源：机构名称 + 报告名称 + 发布日期 + 页码/表格编号)

示例：2025 年 AI 采用率 88% (来源：McKinsey, State of AI 2025, p.12, Table 3)
```

### 2. 多源验证 (Cross-Validation)

- **关键数据**: 必须≥2 个独立来源交叉验证
  - 市场规模、增长率、渗透率等核心指标
  - 不同来源数据差异>10% 时必须说明原因
  - 优先采用官方统计数据

- **验证等级**:
  - A 级：3 个以上独立来源一致
  - B 级：2 个来源一致
  - C 级：仅 1 个来源 (需标注"待验证")

### 3. 合理性检验 (Sanity Check)

- **异常检测**: 数据必须在合理范围内
  - ROI: 通常 10-50% (超过 100% 必须特别说明)
  - 增长率：通常 -20% 到 +100% (超过需说明)
  - 市场份额：0-100%，总和应≈100%
  - 调查覆盖率：0-100%

- **逻辑自洽**:
  - 部分之和应等于整体
  - 同比/环比计算必须准确
  - 利润率不能超过 100%

### 4. 计算透明 (Calculation Transparency)

- **公式公开**: 所有计算必须展示公式

```markdown
ROI = (收益 - 成本) / 成本 × 100%

其中：
- 收益 = [具体构成]
- 成本 = [具体构成]
- 计算周期 = [时间范围]
```

- **假设明确**: 必须说明关键假设

```markdown
关键假设:
1. 市场规模年增长率：15% (来源：[权威预测])
2. 目标市场份额：3% (基于 [对标企业])
3. 净利率：20% (行业平均 15-25%)
```

- **敏感性分析**: 关键变量±20% 的影响

```markdown
敏感性分析:
- 乐观情景 (+20%): ROI = 35%
- 基准情景：ROI = 25%
- 悲观情景 (-20%): ROI = 15%
```

### 5. 失败条件 (Falsifiability)

- **可证伪性**: 必须说明什么情况下结论不成立

```markdown
本结论在以下情况下不成立:
1. 若 2026 年 AI 采用率<70% (当前 88%)
2. 若能源成本占比>40% (当前 25%)
3. 若监管政策收紧 (如限制 AI 应用)
```

### 6. 数据审计追踪 (Audit Trail)

- **原始数据存档**: 必须保存原始数据文件
- **处理过程可追溯**: 数据处理步骤必须记录
- **版本管理**: 数据更新必须标注版本

## 📊 修正后的评分标准

| 维度 | 权重 | 5 分标准 (真实优先) | 否决项 |
|------|------|------------------|--------|
| **1. 数据真实性** | 20% | 100% 数据可追溯，无造假 | ❌ 发现造假即 0 分 |
| **2. 来源权威性** | 15% | 100% 来自权威机构 | ❌ 使用不可靠来源 |
| **3. 多源验证** | 12% | 关键数据≥2 源验证 | - |
| **4. 合理性检验** | 10% | 数据在合理范围，异常已说明 | ❌ 明显异常未说明 |
| **5. 计算透明** | 8% | 公式、假设、敏感性完整 | - |
| **6. 结构完整性** | 5% | 含封面/目录/正文/附录 | - |
| **7. 逻辑链条** | 8% | 发现→分析→建议完整 | - |
| **8. 框架应用** | 5% | 2 个以上专业框架 | - |
| **9. 洞察深度** | 5% | 有反常识洞察 + 二阶效应 | - |
| **10. 建议质量** | 5% | SMART+ 可执行 | - |
| **11. 可证伪性** | 4% | 明确失败条件 | - |
| **12. 图表质量** | 8% | 图表专业、数据准确 | - |

**综合得分** = Σ(维度得分 × 权重)

**L5-R 达标线**:
- 综合得分 ≥4.5
- **数据真实性必须≥4.5**
- **来源权威性必须≥4.5**
- **无造假记录**

**一票否决**:
- 发现数据造假 → 直接 0 分
- 使用不可靠来源 → 本项 0 分
- 明显异常未说明 → 降 1 级

## 🔍 数据验证流程 (强制)

### 阶段 0: 数据收集

1. 收集数据时同步记录来源
2. 优先选择权威来源
3. 记录数据发布日期和版本

### 阶段 1: 数据验证

1. **来源验证**: 是否来自权威机构？
2. **时效验证**: 数据是否过时？
3. **一致性验证**: 多源数据是否一致？
4. **合理性验证**: 数据是否在合理范围？

### 阶段 2: 计算验证

1. **公式验证**: 计算公式是否正确？
2. **假设验证**: 假设是否合理？
3. **敏感性验证**: 关键变量变化影响？
4. **可复现验证**: 他人能否复现结果？

### 阶段 3: 报告撰写

1. **数据标注**: 每个数据标注来源
2. **假设说明**: 明确关键假设
3. **失败条件**: 说明什么情况下结论不成立
4. **审计追踪**: 保存原始数据和处理过程

## 📝 示例：正确的数据标注

### ❌ 错误示例

```markdown
2025 年 AI 采用率达到 88%，ROI 高达 1150%。

问题：无来源、无计算过程、数据异常
```

### ✅ 正确示例

```markdown
2025 年企业 AI 采用率达到 88% (McKinsey, State of AI 2025, p.12, n=2,800 企业)。

企业 AI 投资平均 ROI 为 25% (McKinsey 2025: 25%; BCG 2025: 23%; Deloitte 2025: 27%)。

头部企业 (top 10%) ROI 可达 40-60% (McKinsey 2025, p.18)。

ROI 计算:
ROI = (年度收益 - 投资成本) / 投资成本 × 100%

其中:
- 年度收益 = 效率提升收益 + 成本节约收益
- 投资成本 = 技术采购 + 实施成本 + 培训成本
- 计算周期 = 12 个月

关键假设:
1. 效率提升：20% (基于 McKinsey 基准研究)
2. 实施周期：6 个月 (行业平均 4-8 个月)
3. 技术成本：年降 15% (Gartner 预测)

敏感性分析:
- 乐观情景 (+20% 效率): ROI = 35%
- 基准情景：ROI = 25%
- 悲观情景 (-20% 效率): ROI = 15%

失败条件:
若 AI 采用率<70% 或 能源成本>40% 总成本，则 ROI 可能<10%。
```

5. **波特五力结构图** - ASCII 图示
6. **SWOT 四象限矩阵** - 优劣势机会威胁对照表

### 图表格式标准

- **趋势图**：使用 ↑↓ 箭头，数量表示强度
- **占比图**：使用 █ 方块条，长度表示比例
- **矩阵图**：使用表格 + 边框字符
- **结构图**：使用层级列表 + 缩进

### 图表质量要求

- ✅ 每个数据表后必须紧跟对应图表
- ✅ 图表必须与数据一致
- ✅ 图表必须清晰可读
- ✅ 禁止使用外部图片（仅用文本字符）
- ❌ 禁止只有文字描述没有图表

## 📄 PDF 生成流程

### 前置要求

**必须安装**：

```bash
sudo apt-get install pandoc texlive-xetex texlive-lang-chinese
```

### 生成步骤

1. **生成 Markdown 报告**

   ```bash
   ./scripts/generate.sh -t "报告主题" -o report.md
   ```

2. **转换为 PDF**

   ```bash
   ./scripts/to_pdf.sh -i report.md -o report.pdf -v
   ```

3. **验证 PDF**
   - 检查中文字体是否正确
   - 检查图表是否完整
   - 检查页码和目录

### 字体配置

- **默认字体**: Noto Sans CJK SC
- **备选字体**: Arial Unicode MS, AR PL UKai CN
- **要求**: 必须支持中文的 Unicode 字体

### 故障排除

**问题 1**: 中文显示为方块

- 原因：字体不支持中文
- 解决：安装 `texlive-lang-chinese`

**问题 2**: 图表字符缺失

- 原因：字体不支持特殊字符
- 解决：使用 Noto Sans CJK SC 或 Arial Unicode MS

**问题 3**: PDF 生成失败

- 检查 pandoc 版本：`pandoc --version`
- 检查 xelatex：`xelatex --version`
- 查看详细错误信息
